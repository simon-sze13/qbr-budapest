# Lab 2 — Deploy workload

## Objectives

- Execute the **core QBR demo**: deploy an application or Helm release through Cloud Factory.
- Show **repeatability**: the same steps your SE team uses in customer POCs (simplified for time).

## Prerequisites

- Completed [Lab 1 — Environment setup](lab-01-environment.md).

## Steps

1. In the UI, navigate to the flow your facilitator defines (for example **Applications**, **Catalog**, or **Deployments**).
2. **Select** the demo application or chart agreed for QBR.
3. **Configure** minimal settings (namespace, release name, values) per script.
4. **Deploy** and wait until the workload reaches a healthy state.

!!! tip "Narration"
    Call out **what would differ** in a customer tenant: credentials, cluster choice, and policy gates—without exposing internal names on screen recordings.

## Expected outcome

- Workload shows **running / healthy** in the UI or equivalent status.
- You can name **one customer benefit** tied to this action (speed, standardization, or governance).
