# Lab 2 — Projects and catalogs

## Goal

Use MCP tools to **navigate** CCF: projects, optional virtual clusters, and **catalog** contents—still within facilitator-approved scope.

## Steps

1. **`list-projects`** — Capture a **project ID** you are allowed to use.
2. Optionally **`list-virtual-clusters`** with the **parent project** your script specifies.
3. **`catalog-list`** — List catalogs; pick a **catalog ID** for read-only inspection.
4. **`catalog-apps-list`** — List applications registered in that catalog.
5. Optional (if allowed): **`available-apps-list`** with a small **limit** to see repository packages.

## Facilitator extensions

- **Write** labs (create project, `app-install`) should only run in **dedicated workshop tenants** with explicit approval.
- Pair with [Exposing applications](https://docs.taikun.cloud/CloudWorks/Exposing_Applications/Exposing_Applications/) docs if the demo includes **Ingress / Taikun Link** after app install.

## Success criteria

- [ ] You can correlate **IDs** from MCP output with what you see in the **CCF UI**.
- [ ] You understand which tools are **read** vs **write** before attempting changes.
